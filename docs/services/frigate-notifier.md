# Frigate Notifier

Small systemd service that subscribes to Frigate MQTT events and sends filtered object notifications by email or ntfy push.

## Status

**Enabled:** No, paused on 2026-04-27 because the event volume was too noisy.

Production keeps the notifier configuration in place but does not start `frigate-notifier.service` while `homelab.services.frigate-notifier.enable = false`. When re-enabled, it is configured in `auto` delivery mode for `person` and `car` events from `camera2`. When the notifier environment file contains `NTFY_TOPIC_URL`, it sends ntfy push notifications with the annotated snapshot as an image attachment and a tap-through link to the Frigate review page. Without `NTFY_TOPIC_URL`, it falls back to the existing email path to `jan@kaifer.cz`.

Email notifications include the existing Frigate event snapshot JPEG attachment plus a second `-annotated.jpg` attachment generated by the notifier with a thick red detection box and red label banner, with the latest camera frame as a fallback source image. ntfy push notifications send the annotated image directly as the notification attachment.

Notifications are intentionally limited to Frigate `new` events. Car notifications also require Frigate to mark the object as active/non-stationary, which prevents a parked car from generating repeated overnight emails when it is re-detected.

## Configuration

**Module:** `modules/services/frigate-notifier.nix`  
**Pattern:** `homelab.services.frigate-notifier.enable`

Important options:

| Option | Default | Description |
|--------|---------|-------------|
| `deliveryMode` | `email` | `email`, `ntfy`, `both`, or `auto` |
| `recipient` | null | Destination email address when email delivery is enabled |
| `labels` | `[ "person" "car" ]` | Object labels that trigger notifications |
| `eventTypes` | `[ "new" ]` | Frigate event types that trigger notifications |
| `activeOnlyLabels` | `[ "car" ]` | Labels that only notify while active/non-stationary |
| `cameras` | `[ "camera2" ]` | Cameras that trigger notifications |
| `topic` | `frigate/events` | Frigate MQTT event topic |
| `cooldownSeconds` | `300` | Per-event notification cooldown |
| `frigateApiUrl` | `http://127.0.0.1:5000` | Local Frigate API used for snapshot attachments |
| `mqttPasswordFile` | required | MQTT password file |
| `smtpEnvironmentFile` | null | SMTP credential environment file |
| `ntfyTopicUrl` | null | ntfy topic URL, if not supplied through the environment file |
| `ntfyEnvironmentFile` | null | Environment file containing `NTFY_TOPIC_URL`, `NTFY_TOKEN`, or `NTFY_AUTH_HEADER` |
| `ntfyPriority` | `4` | ntfy notification priority |
| `ntfyTags` | `[ "camera" ]` | ntfy notification tags |

The annotated attachment is generated from the fetched snapshot using Frigate's MQTT event metadata. If Frigate includes an object `box`, the notifier draws a thick red rectangle around that object and labels it with the object type and confidence. If the event does not include usable box coordinates, the notifier draws a thick red border around the whole image and still labels what was detected.

Example production config:

```nix
homelab.services.frigate-notifier = {
  enable = false;
  deliveryMode = "auto";
  recipient = "jan@kaifer.cz";
  mqttPasswordFile = config.age.secrets.mqtt-frigate-password.path;
  smtpEnvironmentFile = config.age.secrets.frigate-notifier-smtp-env.path;
  ntfyEnvironmentFile = config.age.secrets.frigate-notifier-smtp-env.path;
};
```

## Notification Secret

The production notification secret is `secrets/frigate-notifier-smtp-env.age`. It currently stores the SMTP fallback settings and can also store ntfy settings.

SMTP fallback settings:

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USERNAME=notifier@example.com
SMTP_PASSWORD=change-me
SMTP_FROM=Frigate <notifier@example.com>
SMTP_ENVELOPE_FROM=notifier@example.com
SMTP_STARTTLS=true
```

Use `SMTP_SSL=true` and `SMTP_PORT=465` instead when the provider requires implicit TLS.
For Google Workspace SMTP relay with "Only registered Apps users", keep
`SMTP_ENVELOPE_FROM` set to the authenticated Workspace user even if the visible
`SMTP_FROM` display address is an alias-style notification address.

ntfy push settings:

```env
NTFY_TOPIC_URL=https://ntfy.sh/example-private-topic
NTFY_TOKEN=tk_example
```

`NTFY_AUTH_HEADER` can be used instead of `NTFY_TOKEN` when a custom Authorization header is needed. Once `NTFY_TOPIC_URL` is present, `deliveryMode = "auto"` sends push notifications first and only falls back to email if the ntfy publish request fails.

Operational setup:

1. Install the ntfy mobile app.
2. Subscribe to the private/protected topic.
3. Add `NTFY_TOPIC_URL` and, for protected topics, `NTFY_TOKEN` to `secrets/frigate-notifier-smtp-env.age`.
4. Deploy or restart `frigate-notifier.service`.

## Runtime

Check service status:

```bash
systemctl status frigate-notifier
```

Check logs:

```bash
journalctl -u frigate-notifier -n 200 --no-pager
```

## Dependencies

- Frigate MQTT publishing
- Mosquitto
- Working outbound SMTP credentials for email fallback
- ntfy topic access for push delivery
